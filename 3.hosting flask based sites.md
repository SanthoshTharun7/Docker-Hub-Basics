1.Today's class focused on uploading various versions of a file in the same repo.

version: 1.0 consists of hosting a site containing directory consisting of few text files.

```
docker build . < Dockerfile -t tag

docker build -t tag .

docker tag (tag name) username/reponame with version

docker run -d -p 80XX-(exposed port no.)  --name (custom container name) tag

docker image push -a user/repo name with version
```

Scenario 1:

if for example  version is running on 8041 port and you want to host version 2 in the very same port:


1.check for docker status (shows details about container and tags):

```
docker ps
```

2.remove the container using the port:

```
docker rm -f (continer name)
```

3. again check with 'docker ps' cmd to check allocation status.

4. you may create a new folder for version: 2.0, this may contain pdf, image files.

```
docker build -t tag .
```

above command updates the docker details in newly created folder.

5.attach the tag with repo (version to be mentioned without fail).

```
docker tag  sourceimage targetimage
```
#source image (stored locally), target image is the repo where source to be pushed.

6.Run the local host website.
```
docker run -d -p 80xx:exposed port targetimage
```
-d : detaches container operation from occupying terminal (works in background)

-p: allocation of port from host and container (host:container) <-- format.

7.Push the image.

```
docker image push -a targetimage
```
